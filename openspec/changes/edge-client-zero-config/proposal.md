# Edge Client 零配置启动

## Summary

为 Edge Client 实现零配置启动功能，让小白用户只需运行 `edge-client start` 即可开始使用，同时保留开发人员的高级配置能力。

## Motivation

### 当前问题

1. **必须手动编写配置文件**：用户需要了解 YAML 格式
2. **必须指定 --config 参数**：增加使用门槛
3. **需要知道设备凭证**：不知道从哪里获取
4. **技术门槛高**：对非技术用户不友好

### 用户痛点

**小白用户**：
- "我只想让摄像头推流，为什么这么复杂？"
- "配置文件怎么写？"
- "设备 ID 和密钥在哪里？"

**开发人员**：
- 希望保留灵活的配置能力
- 需要支持自动化部署
- 需要命令行参数覆盖

## What Changes

### 目标用户体验

#### 小白用户（零配置）

```bash
# 首次运行
$ ./edge-client start

⚙️  未检测到配置文件，启动设置向导...
🌐 Web UI 已启动: http://localhost:8088/setup
🔗 正在打开浏览器...

[用户在浏览器中完成配置]

✅ 配置已保存！
🚀 启动推流...
```

#### 开发人员（高级配置）

```bash
# 使用自定义配置
./edge-client start --config /path/to/config.yaml

# 使用命令行参数
./edge-client start --device-id xxx --device-secret yyy

# 组合使用
./edge-client start --config my.yaml --input-type rtsp
```

### 核心功能

1. **自动配置文件查找**
   - 优先级：`./config.yaml` > `./configs/config.yaml` > `~/.pawstream/config.yaml`
   - `--config` 参数变为可选

2. **设置向导模式**
   - 检测配置完整性
   - 自动进入向导模式
   - Web UI 引导配置
   - 自动生成配置文件

3. **配置完整性检查**
   - 必需字段：device.id, device.secret, api.url
   - 可选字段：使用智能默认值

4. **命令行参数覆盖**
   - 优先级：命令行 > 配置文件 > 默认值
   - 支持所有核心配置项

## Risks and Mitigations

### 风险

1. **配置文件位置冲突**
   - 风险：多个配置文件存在
   - 缓解：明确的优先级规则，日志提示

2. **设置向导复杂度**
   - 风险：开发时间长
   - 缓解：分阶段实施，先简后繁

3. **用户误操作**
   - 风险：配置错误导致无法启动
   - 缓解：配置验证，回滚机制

### 缓解措施

- 详细的日志输出
- 配置备份机制
- 错误提示友好化
- 提供配置模板
