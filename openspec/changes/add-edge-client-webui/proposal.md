# Change: 为边缘客户端添加 Web UI 管理界面

## Why

边缘推流客户端目前需要通过配置文件（YAML）或命令行进行配置和管理，这对非技术用户存在较高门槛。特别是在两种常见场景下：

1. **预装设备场景** - 客户端烧录到操作系统中，随系统自启动，用户首次使用时需要配置设备信息
2. **分发场景** - 普通用户获得编译好的客户端，需要简单的配置方式即可使用

现有方式要求用户：
- 了解 YAML 配置格式
- 知道设备 ID 和密钥如何获取
- 理解各种配置项的含义
- 使用命令行或文本编辑器
- SSH 连接到设备查看日志

这导致小白用户难以独立完成配置和日常管理，需要技术支持。

## What Changes

添加内嵌的 Web UI 管理界面，随客户端启动，提供配置、监控、日志查看的统一界面：

### 核心功能
- **配置管理** - 可视化编辑所有配置项，保存后自动生效（无需重启）
- **设备绑定** - 通过 Web UI 登录用户账号，选择或创建设备，自动获取设备 ID 和密钥
- **状态监控** - 实时显示推流状态、系统资源、连接信息
- **日志查看** - 滚动显示最近日志，便于调试
- **配置热重载** - 使用 fsnotify 监控配置文件变化，自动重新加载
- **纯静态实现** - HTML/JS/CSS，无需额外依赖，嵌入到 Go 二进制

### 技术方案
- 使用 Go 标准库 `net/http` 提供 HTTP 服务器
- 使用 `embed` 打包静态资源（HTML/JS/CSS）
- 使用 `fsnotify` 监控配置文件变化，实现热重载
- 提供 RESTful API + WebSocket 供前端调用
- 前端使用 Vanilla JS（无框架依赖）

### Web UI 界面结构
```
┌─────────────────────────────────────┐
│  🐾 PawStream 边缘客户端              │
├─────────────────────────────────────┤
│  [配置] [状态] [日志]  ← 标签页       │
├─────────────────────────────────────┤
│  【配置】- 可视化编辑配置项            │
│  【状态】- 实时推流状态和系统信息      │
│  【日志】- 最近日志滚动显示            │
└─────────────────────────────────────┘
```

### 命令简化
```bash
edge-client start              # 启动推流 + Web UI（默认端口 8080）
edge-client start --daemon     # 后台运行 + Web UI
edge-client start --no-webui   # 仅推流，不启动 Web UI
edge-client stop               # 停止
edge-client restart            # 重启
edge-client status             # 命令行快速查看状态
```

**移除的命令**：
- `setup` - 不再需要，Web UI 一直可访问
- `dashboard` - 不再需要，Web UI 就是管理界面

### Web UI 生命周期
- **默认行为** - 随 `start` 命令启动，监听端口 8080
- **持续运行** - 推流过程中 Web UI 一直可访问
- **资源消耗** - HTTP 服务器空闲时仅占用 ~5MB 内存
- **访问方式** - 局域网任何设备通过 `http://设备IP:8080` 访问

### 配置热重载机制
- 使用 `fsnotify` 监控 `config.yaml` 文件变化
- 任何方式修改配置（Web UI 保存 或 手动编辑）都会触发热重载
- 检测到变化后，优雅停止当前推流，重新加载配置，重启推流
- 整个过程无需重启客户端进程
- Web UI 通过 WebSocket 实时更新状态

### 设备发现
- 固定端口 8080
- 日志输出访问地址（如 `http://192.168.1.100:8080`）
- 后期可扩展：mDNS 广播（`pawstream-edge.local`）

### 安全性
- **局域网访问** - 默认监听所有接口，假设局域网可信
- **可选认证** - 可配置 HTTP Basic Auth（用户名/密码）
- **审计日志** - 记录所有配置修改操作
- **只读操作** - 状态和日志查看是只读的，不影响系统

## Impact

### 受影响的组件
- **edge-client** - 主要变更
  - 新增 `internal/webui/` 模块（HTTP 服务器、API 处理器、静态资源、WebSocket）
  - 新增 `web/` 目录（HTML/JS/CSS 源文件）
  - 修改主程序默认启动 Web UI
  - 新增 `fsnotify` 配置文件监控机制
  - 移除 `setup` 和 `dashboard` 子命令

### 受影响的规格
- `edge-client` - 新增 Web UI 管理界面相关需求

### 用户体验改进
- ✅ 小白用户可以通过浏览器完成所有配置
- ✅ 可视化配置流程，降低出错率
- ✅ 实时查看推流状态和日志，无需 SSH
- ✅ 配置修改后自动生效，无需重启
- ✅ 无需学习 YAML 语法和命令行
- ✅ 符合现代物联网设备使用习惯（如路由器、NAS）

### 技术债务
- 新增 Go HTTP 服务器代码（~1000 行）
- 新增 fsnotify 依赖
- 新增前端静态资源（预计 < 100KB）
- 二进制文件大小增加约 200-300KB

### 风险
- **极低风险** - Web UI 可通过 `--no-webui` 选项禁用
- **资源可控** - HTTP 服务器资源消耗极小（~5MB 内存）
- **安全可控** - 局域网隔离 + 可选认证
- **兼容性** - 保持向后兼容，配置文件格式不变
- **热重载稳定性** - fsnotify 是成熟的库，MediaMTX 等项目已验证
