# Change: 为边缘客户端添加 Web UI 配置界面

## Why

边缘推流客户端目前需要通过配置文件（YAML）或命令行进行配置，这对非技术用户存在较高门槛。特别是在两种常见场景下：

1. **预装设备场景** - 客户端烧录到操作系统中，随系统自启动，用户首次使用时需要配置设备信息
2. **分发场景** - 普通用户获得编译好的客户端，需要简单的配置方式即可使用

现有方式要求用户：
- 了解 YAML 配置格式
- 知道设备 ID 和密钥如何获取
- 理解各种配置项的含义
- 使用命令行或文本编辑器

这导致小白用户难以独立完成配置，需要技术支持。

## What Changes

添加内嵌的 Web UI 配置界面，提供用户友好的首次配置向导：

### 核心功能
- **首次配置向导** - 引导式配置流程（API 服务器 → 用户登录 → 设备绑定 → 视频源选择）
- **设备绑定** - 通过 Web UI 登录用户账号，选择或创建设备，自动获取设备 ID 和密钥
- **配置生成** - 根据 Web UI 输入自动生成 `config.yaml`
- **状态仪表盘** - 显示客户端运行状态、推流状态、日志预览
- **纯静态实现** - HTML/JS/CSS，无需额外依赖，嵌入到 Go 二进制

### 技术方案
- 使用 Go 标准库 `net/http` 提供 HTTP 服务器
- 使用 `embed` 打包静态资源（HTML/JS/CSS）
- 提供 RESTful API 供前端调用
- 前端使用 Vanilla JS（无框架依赖）

### 子命令
```bash
edge-client setup     # 进入配置模式（启动 Web UI）
edge-client start     # 正常运行模式（如果未配置，自动进入 setup）
edge-client dashboard # 启动状态仪表盘
```

### 配置模式
- **未配置时** - 自动进入配置模式，启动 Web UI（端口 8080）
- **已配置时** - 正常启动推流，Web UI 关闭
- **重新配置** - 使用 `edge-client setup` 命令重新进入配置模式

### 设备发现
- 固定端口 8080
- 日志输出访问地址（如 `http://192.168.1.100:8080`）
- 后期可扩展：mDNS 广播（`pawstream-edge.local`）

### 安全性
- **配置模式** - 在局域网内开放访问（假设局域网可信）
- **临时访问码**（可选）- 首次启动生成 6 位数字码，访问 Web UI 需要输入
- **配置完成后** - Web UI 自动关闭，仅在运行模式下开放 dashboard（需要认证）

## Impact

### 受影响的组件
- **edge-client** - 主要变更
  - 新增 `internal/webui/` 模块（HTTP 服务器、API 处理器、静态资源）
  - 新增 `web/` 目录（HTML/JS/CSS 源文件）
  - 修改主程序支持 `setup` 和 `dashboard` 子命令
  - 修改配置逻辑，检测是否已配置

### 受影响的规格
- `edge-client` - 新增 Web UI 配置相关需求

### 用户体验改进
- ✅ 小白用户可以通过浏览器完成配置
- ✅ 可视化配置流程，降低出错率
- ✅ 实时反馈配置状态
- ✅ 无需学习 YAML 语法和命令行

### 技术债务
- 新增 Go HTTP 服务器代码
- 新增前端静态资源（预计 < 50KB）
- 二进制文件大小增加约 100-200KB

### 风险
- **低风险** - Web UI 是可选功能，不影响现有 CLI 配置方式
- **兼容性** - 保持向后兼容，配置文件格式不变
